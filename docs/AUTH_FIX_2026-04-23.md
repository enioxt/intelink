# Auth System Root Cause Analysis & Fix (2026-04-23)

## Issue Summary
Users were successfully logging in via email/password, but after form submission, were redirected back to the landing page instead of reaching the dashboard at `/central`.

## Root Cause
The authentication system has two interconnected parts:
1. **Supabase Authentication** - handles email/password auth
2. **App v2 Session System** - manages JWT tokens and sessions with cookies

The `/api/auth/bridge` endpoint is supposed to bridge between them, but was **not creating a JWT session or setting the required auth cookies**.

### Detailed Flow of the Bug
1. User fills login form and submits
2. `handleEmailLogin` calls `supabase.auth.signInWithPassword({email, password})`
   - ✓ Creates Supabase session
3. Then calls `bridgeMember(email)` → `POST /api/auth/bridge`
   - ✗ Old behavior: Returns member data but NO JWT session or cookies
4. `router.push(returnUrl)` redirects to `/central`
5. When `/central` loads, `app/page.tsx` mounts
6. `checkAuth` useEffect runs and calls `/api/v2/auth/verify`
   - This endpoint calls `getAccessTokenFromRequest()` which looks for `intelink_access` cookie
7. **BUG**: No `intelink_access` cookie exists (bridge never set it)
8. `/api/v2/auth/verify` returns 401 "No access token"
9. `app/page.tsx` sees 401, clears all localStorage
10. Sets `isAuthenticated = false`
11. Renders `<PublicLanding />` (landing page) instead of dashboard

**Result**: User is redirected to landing page after successful login!

## Solution
Modified `/api/auth/bridge/route.ts` to properly bridge Supabase auth to app session:

```typescript
// 1. Create JWT session in app's auth system
const sessionResult = await createSession({
    member: memberData,
    deviceInfo: {
        userAgent: request.headers.get('user-agent') || 'Supabase Bridge',
        ip: request.headers.get('x-forwarded-for')?.split(',')[0],
    },
});

// 2. Set JWT session cookies (required for /api/v2/auth/verify to work)
if (sessionResult.accessToken && sessionResult.refreshToken) {
    setAuthCookies(response, sessionResult.accessToken, sessionResult.refreshToken);
}
```

### What Gets Set
- `intelink_access` cookie: JWT access token (8 hours expiry)
- `intelink_refresh` cookie: JWT refresh token (7 days expiry)
- `intelink_verified` cookie: Set if member.verified_at exists (verification gate)

## New Flow (Fixed)
1. ✓ Supabase signInWithPassword → Supabase session
2. ✓ `/api/auth/bridge` → Creates JWT session + Sets auth cookies
3. ✓ `router.push('/central')` → Navigates to dashboard
4. ✓ `app/page.tsx` → Calls `/api/v2/auth/verify`
5. ✓ `getAccessTokenFromRequest()` → Finds `intelink_access` cookie
6. ✓ Session verified → `isAuthenticated = true`
7. ✓ Dashboard renders instead of landing page
